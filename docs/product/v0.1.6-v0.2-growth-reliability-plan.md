Target releases: v0.1.6 trust patch, v0.2.0 monitoring and growth release  
Coding loop: NOT APPROVED until a slice is explicitly selected

## 1. Executive decision

MoneySiren already has a useful local-first security model, read-only connectors,
fixture-backed demos, a CLI, a web dashboard, and a desktop HUD. The next release
must not expand the provider count. It must first make the existing product
trustworthy, measurable, installable, and easy to contribute to.

Delivery is split into three horizons:

1. v0.1.6 trust patch
   - replace the unofficial Codex reset-credit path with the official Codex App
     Server interface;
   - make GPT-5.6 token and model reporting explicit and safe;
   - fix sync exit codes and stale-data reporting;
   - restore release, version, and install consistency;
   - add cross-platform source-free install gates;
   - reopen the public contribution path.
2. v0.2.0 monitoring and growth release
   - add an opt-in local scheduler;
   - deliver real desktop and Slack notifications;
   - reduce first-value time with a guided mock demo;
   - publish a static documentation site and a concise README;
   - establish maintainer and contributor operating rules.
3. v0.2.x long-running reliability
   - split the local AI monolith;
   - add SQLite retention and bounded queries;
   - harden process lifecycle.

New providers, OAuth, hosted SaaS, multi-user mode, enterprise FinOps, and
provider write actions remain out of scope.

## 2. Product and security boundaries

- CLI-first and local SQLite remain the default.
- Provider connectors remain read-only.
- Raw provider payloads are never persisted.
- MoneySiren must not return, log, persist, fixture, or screenshot:
  - local AI prompt or assistant text;
  - shell command bodies;
  - raw JSONL lines;
  - Codex or Claude auth files;
  - access or refresh tokens;
  - account, organization, project, invoice, or billing identifiers.
- Codex subscription credits and OpenAI Platform API dollars remain separate
  domains.
- Provider-reported quota, reset, and cost values outrank local estimates.
- Every estimate exposes source, freshness, and accuracy.
- Telemetry remains off. Growth measurement uses public GitHub/npm signals and
  maintainer-run smoke evidence.
- Scheduled network collection requires explicit local enablement.
- Stable release assets fail closed when integrity or signing requirements are
  not met.

## 3. Release success measures

### v0.1.6 gates

- Production code contains no ChatGPT wham endpoint and no direct Codex
  auth.json reader.
- Codex quota and reset-credit fields come from account/rateLimits/read.
- availableCount remains authoritative when detail rows are capped.
- GPT-5.6 Sol, Terra, and Luna are separately identified.
- gpt-5.6 and gpt-5.6-sol normalize to the same canonical model.
- Token components and total_tokens are not double-counted.
- Truncated local log coverage is visibly marked bounded.
- OpenAI Platform cached input tokens are normalized separately.
- Every provider sync error returns a non-zero CLI exit code.
- Old usable data remains visible after refresh failure but is marked stale.
- npm version, Git tag, GitHub Release, release notes, runtime manifest, and
  CHANGELOG version match.
- Windows, macOS, and Linux source-free mock install smokes pass before latest.
- A non-collaborator can open a repository issue using a safe Issue Form.

### v0.2.0 gates

- An explicitly enabled local scheduler evaluates providers and rules at a
  bounded interval.
- Quiet hours, cooldown, retry, and restart deduplication work.
- Desktop and Slack delivery report success and failure truthfully.
- A clean supported machine reaches the credential-free dashboard in five
  minutes.
- README, install docs, static site, and smoke script share one command contract.
- At least five real issues carry good first issue and help wanted labels.
- Governance, triage, security reporting, and release ownership are documented.

## 4. Source-of-truth architecture

| Domain | Authoritative source | Fallback | UI rule |
|---|---|---|---|
| Codex quota/reset | Codex App Server account/rateLimits/read | unavailable; never internal HTTP | official |
| Codex reset credits | App Server rateLimitResetCredits | unavailable; never fabricated rows | official |
| Codex daily/lifetime tokens | App Server account/usage/read | unavailable | official aggregate |
| Codex model/token breakdown | sanitized session metadata | unavailable | estimated or bounded |
| OpenAI Platform cost | organization Costs API | last known safe snapshot | provider reported |
| OpenAI Platform model tokens | organization Usage API | last known safe snapshot | provider reported |
| Provider freshness | sync-run record plus snapshot time | none | live, stale, partial, error |
| Notification decision | normalized local data and preferences | none | local-only |

Official aggregate values and local estimates are displayed separately and are
never summed into a synthetic total.
